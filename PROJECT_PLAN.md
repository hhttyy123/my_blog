# 个人博客项目方案

## 项目概述

搭建一个自由度较高的个人博客网站，主要用于发布文章和展示项目经验，计划使用 XShell 和 XFTP 进行部署。

---

## 技术栈

| 模块 | 技术 | 说明 |
|------|------|------|
| 后端 | Node.js + Express | JavaScript全栈，中间件丰富，社区活跃 |
| 数据库 | SQLite | 零配置，单文件存储，备份简单 |
| 前端 | 纯 HTML/CSS/JS | 渐进式，易扩展，可后续升级为 Vue.js |
| 部署 | Docker | 环境一致，一键部署，易于维护 |

## Docker 部署方案

### 整体架构

```
┌─────────────────────────────────────┐
│           云服务器                   │
├─────────────────────────────────────┤
│  ┌──────────────┐                   │
│  │   Nginx容器  │ ← 处理80/443端口   │
│  │  (反向代理)  │                   │
│  └──────┬───────┘                   │
│         │                           │
│  ┌──────▼───────┐                   │
│  │  应用容器    │ ← Express + 静态文件│
│  │  (Node.js)   │                   │
│  └──────┬───────┘                   │
│         │                           │
│  ┌──────▼───────┐                   │
│  │ SQLite数据   │ ← 数据卷挂载      │
│  │    文件      │   持久化存储      │
│  └──────────────┘                   │
└─────────────────────────────────────┘
```

### 优势

| 好处 | 说明 |
|------|------|
| 环境一致 | 本地测试好，服务器就不会出问题 |
| 一键部署 | `docker-compose up` 搞定 |
| 易于更新 | 停容器→换镜像→启动 |
| 数据安全 | 数据卷挂载到宿主机，删容器不丢数据 |
| 隔离性好 | 不影响服务器其他服务 |

### 需要准备的内容

1. **Dockerfile** - 定义应用如何构建成镜像
2. **docker-compose.yml** - 编排多个容器，配置端口映射、数据卷
3. **nginx.conf** - 反向代理配置，静态文件缓存，SSL配置
4. **数据卷挂载** - SQLite数据库文件映射到宿主机

### 部署流程

```
本地开发完成
    ↓
打包项目代码
    ↓
XFTP 上传到服务器
    ↓
XShell 连接服务器
    ↓
docker-compose build
    ↓
docker-compose up -d
```

### 更新流程

```
XFTP 上传新代码
    ↓
docker-compose build
    ↓
docker-compose up -d
```

### 数据备份

定期把挂载的数据卷目录打包下载即可
