# 1.RAG基本原理
## 检索增强生成

-  **基本流程**：数据清洗加分块—>向量化—>存入向量库—>建立索引—>实时交互
- **应用的工具**：ChromaDB+LangChain
- **ChronmaDB**:向量数据库，存放向量化的数据
- **LangChain**:开发框架，用于链接各种模型，工具，概念，逻辑等
# 2.Hybrid Search
## 混合检索
- 同时使用两种检索方式：关键词检索+RAG
- 根据两种返回结果进行**归一化**处理
- 弥补了关键词检索**不懂语意**，和RAG**过度脑部**的缺点
